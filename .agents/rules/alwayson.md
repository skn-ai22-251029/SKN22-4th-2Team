1. 보안 최우선 (Zero Hardcoding)

OpenAI API Key, AWS Credentials, DB 비밀번호 등 어떠한 민감 정보도 코드에 절대 하드코딩하지 마세요.

모든 환경 변수는 .env 파일(Git 추적 제외)에서 불러오도록 작성하거나, AWS Secrets Manager를 사용하도록 구성하세요.

2. 코드 품질 및 명확성

Python 코드 작성 시 반드시 Type Hints를 적용하고, PEP8 컨벤션을 준수하세요.

RAG 파이프라인과 외부 API 호출부에는 반드시 예외 처리(try-except)와 로깅(logging)을 추가하여 에러 추적이 가능하게 하세요.

코드를 제안할 때는 전체 파일을 다시 쓰지 말고, 수정이 필요한 핵심 부분만 명확히 제시하거나 에디터 도구를 사용해 직접 패치하세요.

3. 도커 및 인프라 표준

Dockerfile은 항상 멀티 스테이지 빌드(Multi-stage build)를 고려하여 이미지 크기를 최적화하고, 불필요한 캐시를 정리하세요.

컨테이너는 루트 권한이 아닌 최소 권한 사용자(non-root user)로 실행되도록 구성하세요.

4. 소통 및 보고 (Communication)

모든 주석은 '한국어'로 작성하세요.

할당된 태스크(Task)가 완료되면, 인박스(Inbox)나 대화창에 "1) 완료한 작업 내역, 2) 다음 단계 권장 사항, 3) PM 에이전트에게 전달할 상태 업데이트 요약"을 간결한 불릿 포인트로 보고하세요.
