### 🛠️ 코드베이스 분석 결과
사용자가 접근하려는 IP(`3.39.228.181`)에서 접속이 되지 않는 원인을 분석한 결과, 다음과 같은 복합적인 구조적 결함이 발견되었습니다.

1. **정적 파일 서빙 누락**: FastAPI 백엔드(`main.py`)가 API 엔드포인트만 제공하고, `frontend/` 폴더의 HTML/JS 파일을 서빙하도록 설정되어 있지 않았습니다.
2. **도커 빌드 컨텍스트 누락**: `Dockerfile`에서 정적 UI 파일이 포함된 `frontend/` 디렉토리를 복사하지 않아, 컨테이너 내부에 UI가 존재하지 않았습니다.
3. **API 주소 하드코딩**: `app.js`에서 API 주소가 `localhost:8000`으로 고정되어 있어, 외부 배포 시 통신이 불가능한 상태였습니다.
4. **접속 포트 불일치**: 앱은 8000번 포트에서 구동 중이나, 사용자는 80번 포트(도메인 기본)로 접속을 시도하여 `Connection Timeout`이 발생하고 있습니다.
