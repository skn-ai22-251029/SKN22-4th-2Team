# [2026-02-27] 프론트엔드 정적 웹 리소스 접속 에러 해결 (UI 복구)

## 🛠️ 코드베이스 분석 결과

사용자님께서 스크린샷으로 보여주신 `{"message":"Welcome..."}` 메시지는 다행히 백엔드 서버 자체의 503 에러가 완벽히 고쳐지고 **정상적으로 서버가 올라와 구동되고 있음(200 OK)을 증명하는 화면**이었습니다! 하지만 기대하던 예쁜 웹 UI가 보이지 않고 JSON 데이터만 출력되는 새로운 문제(UI 접속 에러)가 발생했습니다.

**[문제 원인]**
1. 이전에 백엔드 앱을 `/main.py`에서 `/src/api/main.py` 중심 아키텍처로 리팩토링하는 과정이 있었습니다.
2. 기존 구버전 서버에는 HTML, JS 문서를 제공하는 **정적 파일 마운트 기능**(`app.mount("/", StaticFiles(directory="frontend"))`)이 존재했습니다.
3. 그러나 신규 서버 파일인 `src/api/main.py`을 작성할 때, 백엔드 API 서빙에만 집중하느라 이 웹 정적 리소스 서빙 라우터 부분이 통째로 누락되었습니다.
4. 그 결과 사용자가 루트 주소(`/`)로 접속하면 프론트엔드 HTML(`index.html`) 대신, 직전에 ALB 헬스체크용 픽스로 급하게 추가했던 JSON 응답(`{"message": ...}`)만 달랑 반환되고 말았습니다.

### 조치 내역 (배포 완료)
- `src/api/main.py` 라우터 하단에 **FastAPI 프론트엔드 정적 파일 서빙 기능** 요소들을 다시 복구했습니다.
  - 루트 경로(`/`) 핸들러: `FileResponse("frontend/index.html")` 를 반환하여 첫 방문 시 올바른 HTML 화면을 표시하도록 교체했습니다. (이는 ALB 헬스체크 기준인 HTTP 200 통과 조건도 동시에 완벽히 만족시킵니다.)
  - 애플리케이션 마운트: `app.mount("/", StaticFiles(directory="frontend"), name="frontend")` 를 추가하여 `app.js` 나 CSS 리소스들이 정상적으로 로딩될 수 있도록 배치를 완료했습니다.
- 현재 해당 코드(`4c18e6c`)를 `develop`에 Push 하였으며, 배포 중에 있습니다.
