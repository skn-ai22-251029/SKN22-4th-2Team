### 🛠️ 프론트엔드 코드베이스 분석 결과
(현재 UI 코드의 문제점 및 반응성/접근성 개선 방향 요약)
현재 핵심 기능인 유사 특허 분석(RAG 모델 추론) 실행 시, 10~30초가량의 긴 API 대기 시간이 발생합니다. 이 시간 동안 사용자에게 인터랙티브한 피드백이 제공되지 않는다면 이탈률이 급증하고 서비스 신뢰도가 하락할 수 있습니다.
이를 개선하기 위해 (1) API 호출 즉시 단계별 메시지("검색 중", "분석 중", "생성 중")를 표시하는 프로그레스 바나 스텝퍼 UI를 도입해야 하고, (2) 화면이 멈춘 것처럼 보이지 않도록 동적인 스켈레톤(Skeleton) 및 애니메이션 로딩 인디케이터를 적용해야 합니다. 또한 (3) 긴 대기시간 중 사용자가 잘못 클릭했을 때를 대비한 '분석 취소' 기능을 추가하여 사용자 통제권을 부여하고, (4) 긴 응답 시간의 효율적인 처리를 위해 일반 HTTP Polling 방식보다는 Backend와 Server-Sent Events(SSE) 또는 WebSocket으로 스트리밍 연동하는 방식을 도입해야 합니다.

### 📋 PM 및 Backend 전달용 백로그 (복사해서 각 에이전트에게 전달하세요)
- **Epic: UI 컴포넌트 고도화 (Frontend)**
  - [ ] RAG 대기 화면용 스켈레톤(Skeleton UI) 및 커스텀 애니메이션 로딩 컴포넌트 개발
  - [ ] 단계별 진행 상태(검색 중 → 분석 중 → 생성 중)를 시각화하는 동적 프로그레스(Stepper) UI 구현
  - [ ] (결과 화면 전환 전) 예상 소요 시간 표기 컴포넌트 적용
  - [ ] 실시간 분석 취소(Cancel) 버튼 UI 배치 및 프론트엔드 단의 상태 초기화 로직 작성
- **Epic: 백엔드 API 통신 연동 (Frontend)**
  - [ ] 긴 대기시간 및 단계별 상태 수신을 위한 SSE(Server-Sent Events) 비동기 통신 클라이언트(EventSource) 로직 구현
- **Epic: 백엔드 협업 요청 (Backend에게 전달할 사항)**
  - [ ] 분석 진행 상태 단계별 메시지(Status Code / Message)를 프론트엔드로 스트리밍해 주는 SSE 엔드포인트 구현 요청
  - [ ] 클라이언트에서 '분석 취소(Cancel)' 요청 시, 서버 측의 무거운 LLM 추론 작업을 안전하게 중단(Abort)하는 로직 마련 요청
